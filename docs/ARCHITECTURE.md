# Architecture Design Notes

## Pipeline Overview

This processor implements a **4-stage in-order pipeline**:

```
Clock:   1    2    3    4    5    6    7
Inst A:  IF   ID   EX   WB
Inst B:       IF   ID   EX   WB
Inst C:            IF   ID   EX   WB
Inst D:                 IF   ID   EX   WB
```

> Note: The Memory access and Write-Back stages share one clock cycle in this design.

---

## Stage Breakdown

### Stage 1 â€” Instruction Fetch (IF)
- PC is updated every cycle (or stalled on load-use hazard)
- PC sources: `PC+1` / branch target / jump target / return address / FOR loop address
- 8-to-1 mux (`mux8`) selects next PC based on `PCSrc[2:0]`
- Fetched instruction and `PC+1` are latched into **IF/ID pipeline register** (`flopenr`)

### Stage 2 â€” Instruction Decode (ID)
- Opcode and function field extracted from IF/ID register
- Register file read (combinational)
- Immediate sign/zero extended (parameterised `Extender`)
- Branch comparison resolved here using dedicated `Comparator` â†’ **1-cycle branch penalty** only
- Control signals generated by `mainControlUnit`, `pcControl`, `aluControl`
- Hazard unit checks for load-use stall

### Stage 3 â€” Execute (EX)
- ALU performs operation on forwarded operands A and B
- Forwarding muxes (FA, FB) select from: register file / EX-MEM / MEM-WB / WB outputs
- Result latched into **EX/MEM pipeline register**

### Stage 4 â€” Memory Access + Write-Back (MEM/WB)
- Data memory read (`LW`) or write (`SW`) 
- Writeback mux selects ALU result or memory data
- Result written to register file on **negedge clk** (avoids read/write conflict with posedge read)

## Hazard Unit

Located in `datapath.v` as the `hazard` module.

### Forwarding Logic
```verilog
// Forward A
if      (Rs1 == Rd1 && RegWr1 && Rs1 != 0)  FA = 2'b01;  // EX stage result
else if (Rs1 == Rd2 && RegWr2 && Rs1 != 0)  FA = 2'b10;  // MEM stage result
else if (Rs1 == Rd3 && RegWr3 && Rs1 != 0)  FA = 2'b11;  // WB stage result
else                                          FA = 2'b00;  // register file
```
Same logic applies to `FB` for the B operand.

### Load-Use Stall
```verilog
stall = (FA == 2'b01 || FB == 2'b01) && MemREx;
```
When a load (`LW`) is immediately followed by a dependent instruction, the pipeline freezes for one cycle and inserts a bubble (NOP) into the EX stage.

## Control Unit

The controller (`src/controller/controller.v`) is composed of three sub-units:

### `mainControlUnit`
Decodes opcode + func into datapath control signals:
- `Src1`, `Src2`, `DestReg` â€” register file port selects
- `RegWr`, `MemR`, `MemWr`, `WBData` â€” memory and writeback controls
- `SrcB[1:0]` â€” ALU B-operand select (register / immediate / 1 / 0)
- `ExtOP` â€” sign vs zero extension
- `CompSrc` â€” comparator second operand (0 for BEQ, Rs2 for BNE)
- `for_en` â€” hardware loop enable

### `pcControl`
Decides the next-PC source (`PCSrc[2:0]`) and whether to squash the fetched instruction (`kill`).

### `aluControl`
Maps opcode + func field to a 3-bit `ALU_Ctrl` signal for the ALU.

### Pipeline Register Propagation
Control signals for later stages are propagated through `flopenr` registers, one per pipeline boundary. Stall inserts 0s (bubbles) into the control path without touching the datapath registers.

---

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Branch resolved in Decode | Reduces branch penalty from 2 cycles to 1 |
| Negedge register write | Eliminates read-during-write structural hazard in the same cycle |
| Separate Comparator in Decode | Avoids routing branch result through the ALU stage |
| Hardwired R0 = 0 | Standard RISC convention; simplifies hazard logic (skip R0 forwarding) |
| Harvard memory model | Instruction and data memories are independent â€” no structural hazard between IF and MEM stages |
| Hardware FOR loop | Eliminates software decrement + branch overhead for counted loops |
